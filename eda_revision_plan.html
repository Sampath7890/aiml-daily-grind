The 7-step EDA workflow
Every EDA you ever do — on any dataset — follows these 7 steps in order
1
Load and first look
Get a feel for the data. How big is it? What columns exist? What types?
df = pd.read_csv("file.csv")
df.head() · df.shape · df.columns · df.dtypes · df.info()
2
Check missing values
Which columns have gaps? How many? Should you drop or fill them?
df.isnull().sum() · df.isnull().sum() / len(df) * 100
df.dropna() · df.fillna(df.median())
3
Understand distributions
For each numeric column: what is the range, mean, std? Is it normal, skewed, bimodal?
df.describe() · sns.histplot(df["col"]) · sns.boxplot()
4
Detect outliers
Are there extreme values that will break your model? Boxplot shows them clearly.
sns.boxplot(x="category", y="value", data=df)
df[df["Fare"] > df["Fare"].quantile(0.99)]
5
Explore relationships
How do features relate to each other and to the target? Scatter, pairplot, groupby.
sns.scatterplot(x="f1", y="f2", hue="target", data=df)
df.groupby("category")["target"].mean()
6
Check correlations
Which features are most correlated with the target? Which features are redundant?
df.corr() · sns.heatmap(df.corr(), annot=True, cmap="coolwarm")
df.corr()["target"].sort_values(ascending=False)
7
Write down insights + feature decisions
Before modeling: write which features to use, which to drop, which need transformation. EDA is useless if you don't extract conclusions.
# Example insights to write:
# Feature X has 30% missing — fill with median
# Feature Y has outliers — cap at 99th percentile
# Features A and B are 0.95 correlated — keep only A
Most important rule of EDA: Never touch sklearn until you have completed all 7 steps. A model trained on unclean, unexplored data will always underperform. Real data scientists spend 60–70% of project time on EDA and cleaning — not modeling.

📐 Statistics you know — used in EDA
mean = np.mean(x)
Central tendency — where most values cluster
std = np.std(x)
Spread — how far values are from mean. Small std = tight cluster.
correlation (r)
-1 to +1. Close to ±1 = strong. Close to 0 = no relationship.
Normal distribution
68% within 1σ. 95% within 2σ. Values beyond 3σ = likely outliers.
P(A|B) conditional prob.
How does knowing B change the probability of A? Used in feature analysis.

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
sns.set_theme(style="darkgrid")

df = pd.read_csv("titanic.csv")

# Always run these 5 first on any new dataset
print("Shape:", df.shape)
print("\nFirst 5 rows:")
print(df.head())
print("\nData types:")
print(df.dtypes)
print("\nStatistics:")
print(df.describe())
2 Missing values — find and fix Cleaning ▾
# How many missing per column?
print(df.isnull().sum())

# Percentage missing
print((df.isnull().sum() / len(df)) * 100)

# Fix: fill numeric with median, categorical with mode
df["Age"] = df["Age"].fillna(df["Age"].median())
df["Embarked"] = df["Embarked"].fillna(df["Embarked"].mode()[0])

# Verify: should all be 0 now
print("After cleaning:", df["Age"].isnull().sum())
Decision rule: Column missing <10% → fill with median. Missing 10–40% → fill and add a binary "was_missing" column. Missing >40% → consider dropping the column entirely.
3 Distributions — what does each column look like? Visualization ▾
# Distribution of a numeric column
plt.figure(figsize=(12, 4))

plt.subplot(1, 2, 1)
sns.histplot(df["Age"], bins=30, kde=True)
plt.title("Age Distribution")

plt.subplot(1, 2, 2)
sns.histplot(df["Fare"], bins=30, kde=True, color="coral")
plt.title("Fare Distribution")

plt.tight_layout(); plt.show()

# For categorical columns — value counts
print(df["Survived"].value_counts())
print(df["Pclass"].value_counts())
4 Outliers — spot and handle them Cleaning ▾
# Boxplot to spot outliers visually
plt.figure(figsize=(10, 4))
sns.boxplot(x="Pclass", y="Fare", data=df)
plt.title("Fare Outliers by Class")
plt.show()

# Find outliers numerically — 99th percentile
fare_99 = df["Fare"].quantile(0.99)
outliers = df[df["Fare"] > fare_99]
print(f"Outliers above 99th percentile: {len(outliers)}")

# Cap outliers instead of dropping
df["Fare"] = df["Fare"].clip(upper=fare_99)
5 Relationships — feature vs target Visualization ▾
# Groupby — target mean per category
print(df.groupby("Sex")["Survived"].mean())
print(df.groupby("Pclass")["Survived"].mean())

# Scatter of two features colored by target
plt.figure(figsize=(8, 5))
sns.scatterplot(x="Age", y="Fare",
                hue="Survived", data=df,
                palette="Set1", alpha=0.6)
plt.title("Age vs Fare by Survival")
plt.show()
6 Correlation — find important features Statistics ▾
numeric_cols = ["Survived","Pclass","Age","SibSp","Parch","Fare"]
corr = df[numeric_cols].corr()

# Visualize all correlations
plt.figure(figsize=(8, 6))
sns.heatmap(corr, annot=True, fmt=".2f", cmap="coolwarm", center=0)
plt.title("Feature Correlation Matrix")
plt.show()

# Rank features by correlation with target
print("Feature importance (correlation with Survived):")
print(corr["Survived"].abs().sort_values(ascending=False))
7 Write insights — pairplot + conclusions Summary ▾
# Pairplot — everything vs everything
subset = df[["Age","Fare","Pclass","Survived"]].copy()
subset["Survived"] = subset["Survived"].astype("str")
sns.pairplot(subset, hue="Survived", palette="Set1")
plt.show()

Statistics + Concepts
A column has correlation = -0.34 with the target. What does the negative sign mean? Is this a useful feature?
Negative sign: when this feature increases, the target tends to decrease (inverse relationship).
Example: Pclass correlates ~-0.34 with Survived — higher class number (3rd class) = lower survival.

Is it useful? Yes — |0.34| is moderate correlation. Worth including in the model.
Anything with |r| > 0.1 is usually worth trying.
Fare has mean=32 and std=49. What does this tell you about the distribution? Is it likely normal?
Std (49) is larger than mean (32) — this is unusual.
For a normal distribution, most values fall within 3σ = 3×49 = 147 of the mean.
But Fare can't be negative, so the distribution is right-skewed — a few passengers paid very high fares, pulling the mean up.

This is NOT normal distribution — it needs log transformation before modeling.
Two features have correlation = 0.92 with each other. What should you do before modeling?
They are highly correlated (multicollinearity).
Keeping both gives the model redundant information — one is enough.

Action: check which one correlates more strongly with the target. Keep that one. Drop the other.
Or use PCA to combine them (Phase 3 topic).
