## ⚡ What is This?

This is not another list of YouTube links.

This is the **exact day-by-day, week-by-week learning plan** I built for myself as a 3rd-year B.Tech AI & ML student at KMCE, Hyderabad — structured to get from absolute fundamentals to a deployed, real-world AI engineering portfolio that can target **₹15–25 LPA fresher roles**.

Every single day has a topic. Every week ends with something built. Every phase ends with a project.

**No fluff. No theory dumps. Just what to do — every morning when you open your laptop.**

## 📖 Phase Breakdown (Deep Dive)

<details>
<summary><b>🐍 Phase 1 — Foundations (Weeks 1–6)</b></summary>

**Goal:** Build the bedrock. No shortcuts.

| Week | Python | Mathematics | ML Concept |
|------|--------|-------------|------------|
| 1 | Variables, Lists, Dicts | Vectors as lists | What is ML? Supervised vs Unsupervised |
| 2 | Loops, Functions, List Comprehensions | Dot Product | ML Workflow (data → train → predict) |
| 3 | NumPy arrays, Broadcasting | Matrices, `np.dot()` | Cosine similarity from scratch |
| 4 | Pandas DataFrames, Merge, Groupby | Derivatives (concept) | Linear Regression theory |
| 5 | Matplotlib, Seaborn, EDA patterns | Probability basics | `LinearRegression` in scikit-learn |
| 6 | **MINI PROJECT** | — | Full EDA on Titanic (push to GitHub) |

**Daily rhythm:** Python (Mon) · Math (Tue) · Python (Wed) · Math (Thu) · ML (Fri) · Revise (Sat) · Rest (Sun)

By end of Phase 1 you can: write Python fluently, manipulate data with NumPy + Pandas, build and train a linear regression model, and visualize insights from real datasets.

</details>

<details>
<summary><b>🤖 Phase 2 — Core ML (Weeks 7–12)</b></summary>

**Goal:** Build real models. Understand why they work.

| Week | Topic | Key Concept | Hands-on |
|------|-------|-------------|----------|
| 7 | Linear Regression | Cost function, Train/Test split | California Housing |
| 8 | Logistic Regression + KNN | Sigmoid, Accuracy/Precision/Recall/F1 | Iris classification |
| 9 | Decision Trees + Random Forest | Gini impurity, Bagging | Titanic >80% accuracy |
| 10 | Model Evaluation | Cross-validation, GridSearchCV, Bias-Variance | Tune your best model |
| 11 | Unsupervised — K-Means + PCA | Centroid, Dimensionality reduction | Cluster Iris without labels |
| 12 | **KAGGLE MILESTONE** | Full pipeline | House Prices — real public submission |

By end of Phase 2 you can: train and evaluate classification + regression models, tune hyperparameters, understand every evaluation metric, and submit to Kaggle.

</details>

<details>
<summary><b>⚙️ Phase 2.5 — AI Engineer Layer (Weeks 13–15) ✦</b></summary>

**Why this phase exists:** Every real AI/ML job listing (Impressico Hyderabad, Redsage Hyderabad, Janooma) also requires SQL, a web API framework, and LLM API experience. This phase adds exactly those three skills to your toolkit.

| Week | Skill | What You Build |
|------|-------|---------------|
| 13 | **SQL** — SQLite, JOINs, GROUP BY, Python integration | Import Titanic into SQLite, write 5 analytical queries |
| 14 | **FastAPI** — REST APIs, Pydantic, Swagger, Render deploy | Titanic Random Forest served as a live public API |
| 15 | **Claude/OpenAI API + Prompt Engineering** | News Analyzer CLI → wrapped in FastAPI |

> 💬 **Interview answer you now have:** "I deployed a Random Forest model as a REST API using FastAPI, with JSON input/output, auto-generated Swagger docs, and hosted live on Render." That is a real AI engineer answer.

After Phase 2.5 you can apply to: Redsage Prompt Engineer (Hyderabad), FastAPI + ML Engineer roles, AI developer trainee positions.

</details>

## 📊 EDA Framework — The IQVCS Method

The `eda_revision_plan.html` covers a structured approach to Exploratory Data Analysis used across every project in this roadmap.

```
I — Inspect       Shape, dtypes, head/tail, basic info
Q — Quality       Nulls, duplicates, outliers, inconsistencies
V — Visualize     Distributions, correlations, pairplots, heatmaps
C — Correlate     Feature relationships, target correlation, multicollinearity
S — Summarize     Written insights, business questions answered by data
```

Every dataset in this roadmap goes through IQVCS before modelling begins. This framework is what separates a "model that runs" from a "model that makes sense."

## 🗓️ Weekly Rhythm

Every week in this roadmap follows the same structure:

```
Monday     ──  New concept (Python / ML / Deep Learning)
Tuesday    ──  Mathematics behind Monday's concept
Wednesday  ──  Code implementation of the concept
Thursday   ──  Math deepening or second technique
Friday     ──  Hands-on project / dataset task
Saturday   ──  Revision notebook (write 5 Q&A from the week)
Sunday     ──  Rest (non-negotiable — recovery drives retention)
```

**Rule #1:** Git commit every single weekday. Even 5 lines. The streak is the discipline.

**Rule #2:** No time limit on any phase. Go deep. Speed kills understanding.

## 💡 The Philosophy Behind This Roadmap

Most AI/ML learning resources fall into one of two traps:

1. **Too shallow** — "here are 10 YouTube playlists, good luck"
2. **Too academic** — 300 pages of theory before you build anything

This roadmap does something different: **you build something every Friday, and every 6 weeks you have a portfolio project with a live URL.**

The underlying philosophy:

> **Understanding comes from doing, not from watching.** You don't understand gradient descent until you've coded the weight update loop by hand. You don't understand a Transformer until you've built one in PyTorch. This roadmap makes you do both.

There is also a deliberate decision to **not give a fixed timeline.** You will see "6 weeks minimum" everywhere — that's the minimum if you already know some programming. But if you need 10 weeks for Phase 1, take 10. The goal is mastery, not speed. The job market rewards what you can do, not how fast you learned it.
