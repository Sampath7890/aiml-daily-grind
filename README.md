<details>
<summary><b>🐍 Phase 1 — Foundations (Weeks 1–6)</b></summary>

<br/>

No skipping. No "I already know some Python." Trust the process.

| Week | Python | Mathematics | ML Concept |
|------|--------|-------------|------------|
| 1 | Variables, Lists, Dicts | Vectors as lists | What is ML? Supervised vs Unsupervised |
| 2 | Loops, Functions, List Comprehensions | Dot Product | ML Workflow: data → train → predict |
| 3 | NumPy arrays, Broadcasting | Matrices, `np.dot()` | Cosine similarity from scratch |
| 4 | Pandas DataFrames, Merge, Groupby | Derivatives (concept) | Linear Regression theory |
| 5 | Matplotlib, Seaborn, EDA patterns | Probability basics | `LinearRegression` in scikit-learn |
| 6 | **MINI PROJECT** | — | Full EDA on Titanic → push to GitHub |

**The weekly rhythm:** Python (Mon) · Math (Tue) · Python (Wed) · Math (Thu) · ML (Fri) · Revise (Sat) · Rest (Sun)

After 6 weeks you can: write Python without Googling basic syntax, manipulate any dataset with NumPy + Pandas, build a linear regression model, and make visualizations that actually look good. Most importantly — you'll have your first GitHub commit streak going and something real pushed online.

## 🔍 EDA Framework — The IQVCS Method

Every single dataset in this roadmap goes through the same framework before any model gets trained. It's in `eda_revision_plan.html` with full examples.

```
I — Inspect       Shape, dtypes, head/tail, basic sanity checks
Q — Quality       Nulls, duplicates, outliers, weird values
V — Visualize     Distributions, correlations, pairplots, heatmaps
C — Correlate     Feature-feature and feature-target relationships
S — Summarize     Written findings — what does this data actually say?
```

The S step is what most people skip and it's exactly what interviewers ask about. "What did you find in the EDA?" You need an actual answer, not "I ran `.describe()`."

## 😅 Honest Confessions From Someone Actually Doing This

Look, I'm not gonna pretend this is easy. Here's what actually happened:

- **Week 3 NumPy:** I genuinely did not understand broadcasting for like 4 days. I re-read the docs, watched a video, re-read the docs again. It clicked on day 5. You will have this moment. It's normal.

- **First Kaggle submission:** My model scored worse than just predicting the mean. I submitted it anyway and put it on my resume as "first Kaggle competition." No regrets.

- **FastAPI Week:** I spent 2 hours debugging a 422 error that turned out to be a typo in my Pydantic model. Two hours. For a typo. Welcome to engineering.

- **Every Friday project:** Half of them didn't work the way I planned. I shipped them anyway. A broken project on GitHub beats a perfect project that only exists in your head.

The roadmap has no time limit for a reason. If Week 4 takes you 2 weeks — it takes 2 weeks. The goal is that you actually understand it, not that you hit some arbitrary deadline.

## 🗓️ The Weekly Rhythm (It Actually Works)

Same structure every week. No decision fatigue about what to do today.

```
Monday     ──  New concept (Python / ML / Deep Learning)
Tuesday    ──  The math behind Monday's concept
Wednesday  ──  Code it from scratch
Thursday   ──  Second technique or deeper math
Friday     ──  Build something with it (this is non-negotiable)
Saturday   ──  Write 5 Q&A from the week in a revision notebook
Sunday     ──  Rest. Seriously. Rest.
```

**One rule I follow:** Git commit every weekday. Even if it's 5 lines of code and a comment. The streak is the discipline — it forces you to sit down even when you don't feel like it.

## 📚 Resources That Actually Helped Me

### Math & Theory
- [3Blue1Brown — Essence of Linear Algebra](https://www.youtube.com/playlist?list=PLZHQObOWTQDPD3MizzM2xVFitgF8hE_ab) — watch this before anything else, seriously
- [StatQuest with Josh Starmer](https://www.youtube.com/c/joshstarmer) — he explains everything like you're a person, not a PhD student
- [3Blue1Brown — Neural Networks](https://www.youtube.com/playlist?list=PLZHQObOWTQDNU6R1_67000Dx_ZCJB-3pi) — backpropagation finally makes sense
- Krish Naik's ML playlist — very implementation-focused, great for the coding side

### ML & Deep Learning
- [Fast.ai Practical Deep Learning](https://course.fast.ai/) — top-down approach, you build first then understand why
- [HuggingFace NLP Course](https://huggingface.co/course) — the best free resource for transformers and fine-tuning
- [Andrej Karpathy — Neural Networks: Zero to Hero](https://karpathy.ai/zero-to-hero.html) — you build GPT from scratch, unreal content
- Kaggle Learn — free micro-courses, surprisingly good for feature engineering and SQL

### DSA & Interviews
- [Striver's A2Z DSA Course](https://takeuforward.org/strivers-a2z-dsa-course/) — the most structured DSA path I've found
- [NeetCode Roadmap](https://neetcode.io/roadmap) — pattern-based, great for recognizing problem types fast
- LeetCode Medium — the minimum bar for product company interviews, no way around it

### Tools & Docs
- [FastAPI Official Docs](https://fastapi.tiangolo.com/) — genuinely one of the best-written docs in open source
- [Anthropic API Docs](https://docs.anthropic.com/) — Claude API for building with LLMs
- [LangChain Docs](https://python.langchain.com/) — RAG and agent workflows
- [HuggingFace Transformers Docs](https://huggingface.co/docs/transformers/) — your home base for Phase 4 and 5
