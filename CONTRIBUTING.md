# Contributing to aiml-daily-grind 🤝

First off — thanks for even being here. The fact that you want to contribute to this means a lot.

This is a living roadmap. I'm still following it myself, so it's not perfect and it never will be. That's exactly why contributions matter.

---

## What Kind of Contributions Are Welcome

### ✅ Things that genuinely help

- **Better resources** — Found a video, article, or tool that explains something clearer than what's in the roadmap? Add it.
- **Week notebooks** — Jupyter notebooks implementing any week's topic, runnable on Google Colab. These are the most needed right now.
- **Bug fixes** — Broken links, typos, wrong week numbers, outdated library syntax — fix away.
- **Missing topics** — Something important that a ₹15–25 LPA AI/ML role requires and isn't in the roadmap? Open an issue and let's talk.
- **Translations** — Hindi, Telugu, Tamil, or any other language. Indian students deserve resources in their language too.
- **"I followed this" stories** — If you used this roadmap and got an interview, an internship, or just finally understood backpropagation — share it in Discussions. Seriously.

### ❌ Things that don't help

- Paid course links
- Resources you haven't personally used or verified
- Completely restructuring the phase order without opening an issue first (the ordering is intentional)
- Spammy self-promotion disguised as a contribution

---

## How to Contribute

```bash
# 1. Fork the repo
# 2. Create a branch with a clear name
git checkout -b add/week-3-numpy-notebook
# or
git checkout -b fix/broken-link-phase2
# or
git checkout -b improve/dsa-track-resources

# 3. Make your changes

# 4. Commit with a clear message
git commit -m "Add: Week 3 NumPy broadcasting notebook (Colab-ready)"

# 5. Push and open a Pull Request
git push origin your-branch-name
```

Then open a PR and describe what you changed and why. That's it.

---

## Commit Message Format

Keep it simple and clear:

| Prefix | Use for |
|--------|---------|
| `Add:` | New content, notebooks, resources |
| `Fix:` | Typos, broken links, wrong info |
| `Improve:` | Better explanation of existing content |
| `Remove:` | Outdated or irrelevant content |

Example: `Add: Week 7 Linear Regression sklearn notebook`

---

## Opening an Issue

If you don't want to directly contribute but have a suggestion — open an issue. Label it:

- `suggestion` — for new ideas
- `bug` — for broken links or wrong content
- `question` — if something in the roadmap is unclear

---

## Ground Rules

- Be respectful. Everyone here is learning.
- No gatekeeping. A 1st-year student's contribution is just as valid as a senior dev's.
- If you're not sure whether your contribution fits — open an issue first and ask. I'll respond.
